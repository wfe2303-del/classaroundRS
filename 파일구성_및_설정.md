# 파일구성 및 설정

## 폴더 구조
- `index.html` : 메인 진입 파일
- `assets/classaround-logo.png` : 로그인/헤더 로고
- `src/css/styles.css` : 전체 UI 스타일
- `src/js/config.js` : Google Client ID, 시트 원본 ID, 기본 규칙 설정
- `src/js/auth.js` : Google 로그인 및 권한 처리
- `src/js/sheets.js` : Google Sheets API 호출
- `src/js/parsers.js` : 결제자/신청자 파싱 및 매칭 계산
- `src/js/render.js` : 화면 렌더링
- `src/js/exportReport.js` : 정리표 이미지 저장
- `src/js/state.js` : 로컬 상태 저장
- `src/js/utils.js` : 공통 유틸
- `src/js/charts.js` : 차트 렌더링
- `src/js/app.js` : 이벤트 연결 및 앱 시작점

## 직접 넣어야 하는 값
### `src/js/config.js`
아래 값을 실제 Google OAuth 웹 클라이언트 ID로 바꿔 주셔야 합니다.

```js
export const GOOGLE_CLIENT_ID = 'YOUR_GOOGLE_CLIENT_ID.apps.googleusercontent.com';
```

## 동작 방식
1. 로그인 버튼 클릭
2. Google 계정 선택 및 시트 권한 승인
3. 결제자 시트 목록 로드
4. 현재 선택된 탭이 자동 반영
5. 다른 탭을 선택하면 즉시 결제자 데이터 재반영
6. 신청자 파일 업로드 후 매칭 실행

## 실행 전 확인 사항
- Google Cloud Console에서 **OAuth Client ID**를 발급해 두셔야 합니다.
- 실행 주소를 **Authorized JavaScript origins**에 등록해 두셔야 합니다.
- 브라우저에서 `file://` 직접 실행보다 로컬 서버 또는 배포 주소에서 여는 방식이 안정적입니다.
