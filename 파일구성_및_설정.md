# 파일구성 및 설정

## 파일 목록
- `index.html`
  - 화면 진입점입니다.
- `src/css/styles.css`
  - 전체 스타일입니다.
- `src/js/config.js`
  - Google Client ID, 고정 스프레드시트 ID, 열 규칙을 관리합니다.
- `src/js/auth.js`
  - Google 로그인 / 권한 승인 / 로그아웃 처리입니다.
- `src/js/sheets.js`
  - Google Sheets API 호출 모듈입니다.
- `src/js/state.js`
  - 현재 강사 탭 상태와 로컬 저장소 관리입니다.
- `src/js/parsers.js`
  - 결제자/신청자 데이터 파싱과 매칭 계산입니다.
- `src/js/render.js`
  - 표/그래프/UI 렌더링입니다.
- `src/js/charts.js`
  - 도넛 차트 그리기입니다.
- `src/js/exportReport.js`
  - 정리표 PNG 저장입니다.
- `src/js/utils.js`
  - 공통 유틸입니다.
- `src/js/app.js`
  - 전체 실행 흐름과 이벤트 연결입니다.

## 직접 입력하셔야 하는 값
`src/js/config.js`

```js
export const GOOGLE_CLIENT_ID = 'YOUR_GOOGLE_CLIENT_ID.apps.googleusercontent.com';
```

위 값을 실제 Google OAuth 2.0 웹 클라이언트 ID로 바꿔주셔야 합니다.

## 실행 전 준비
1. Google Cloud Console에서 OAuth 클라이언트 ID를 발급합니다.
2. Google Sheets API를 활성화합니다.
3. Authorized JavaScript origins에 실제 접속 주소를 등록합니다.
   - 예: `http://localhost:5500`
   - 예: `https://your-domain.vercel.app`
4. `src/js/config.js`에 Client ID를 입력합니다.

## 실행 방식
이 프로젝트는 `file://` 더블클릭 실행보다 `localhost` 또는 배포 환경에서 쓰는 방식이 안전합니다.

예시:
- VS Code Live Server
- `npx serve`
- Vercel 정적 배포

## 현재 전제
- 결제자 시트는 고정 Google 스프레드시트에서 읽습니다.
- 신청자 데이터는 아직 별도 시트 정보가 없어서 파일 업로드를 유지했습니다.
