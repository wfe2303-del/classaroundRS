- 결제자 데이터는 고정된 Google 스프레드시트에서만 읽도록 변경했습니다.
- 고정 대상 시트:
  - Spreadsheet ID: `1qclrbo3_VG-sSNIqMW4j1juzwP3nq_ZaT-y1z6WLafc`
  - 기본 gid: `1243994268`

## 2. Google 로그인 + 시트 권한 방식 적용
- Google Identity Services 기준으로 로그인 버튼을 붙였습니다.
- 로그인 후 `spreadsheets.readonly` 권한으로 시트 탭 목록과 값 읽기가 가능합니다.
- 결제자 원본은 브라우저에서 직접 Google Sheets API로 읽습니다.

## 3. 강사 탭별 하드코딩 시트 사용
- 모든 강사 탭이 동일한 Google 스프레드시트를 사용합니다.
- 강사 탭마다 어떤 탭을 읽을지 개별 선택/저장할 수 있습니다.
- 즉, 링크는 고정이고 탭 선택만 강사별로 달라질 수 있습니다.

## 5. 신청자 업로드 유지

- 이 부분까지 제거하면 현재 매칭 기능이 성립하지 않습니다.

## 6. 단일 파일 구조 → 모듈 구조 재정리
- HTML / CSS / JS를 역할별로 분리했습니다.
- 인증, 시트 읽기, 상태관리, 렌더링, 파싱, 이미지 저장을 각각 모듈로 분리했습니다.

## 7. UI 정리
- 불필요한 힌트 문구를 제거했습니다.
- 알림/버튼/상태 문구는 존댓말 기준으로 정리했습니다.
- 상단 액션과 카드 구조를 더 단순하게 정리했습니다.

## 8. 유지한 기능
- 강사 탭 추가/삭제
- 신청자 파일 업로드
- 매칭 실행
- 도넛/막대 그래프
- 전화번호 누락 명단
- 정리표 PNG 저장
